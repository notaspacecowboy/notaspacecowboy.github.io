Overview of Mono & Xamarin
The mono framework is an open-source implementation of .NET Framework based on the open standards for the C# language and the CLR.
Xamarin - a subsidiary of Microsoft - is a company founded by the engineers who created Mono. Xamarin is the primary maintainer and commercial sponsor of Mono.
Scripting with Mono: Why we need Mono?
在古早时期，项目都是由单个编程语言开发。程序员需要在性能更优越的底层语言（C, C++）与开发速度更快但效率更低的高级语言之间仔细选择。
编程语言的运行效率由高到低大致如下：
Assembler
Compiled Static, unsafe languages: C / C++
JITed Static, safe languages: C# / Java
以此为分界线，上下的性能差距极大
JITed Dynamic, safe language: IronPython
Interpreted dynamic, safe languages: Python, Javascript, Lua…
虽然性能差距较大，但使用脚本语言可以进行快速开发，而不必像更贴近底层的语言那样，代码库的一点小改动就会引发整个程序的重新编译。

Developers have turned to scripting as a mechanism to balance the best of both worlds, the idea is simple. The engine of the application is developed in C or C++ and usually maintained and developed by the veteran members of a team, while pieces of the UI, dialogs, interaction, or non-performance critical routines are written in a higher-level scripting language.
这样的开发架构有很多好处，首先开发时我们对上层脚本语言开发者隐藏了由C/C++所编写的引擎的一些底层细节，例如资源管理、内存管理、垃圾回收等，使脚本开发者可以专注于上层功能的实现；另外，脚本开发者在修改或开发脚本代码时不会引发底层代码的重编译。
What Mono Offers?
Mono就给我们提供了这样一套脚本系统的开发环境，使开发者可以使用包括C#在内的一系列语言来扩展我们的应用。
通常意义上的脚本语言都是解释型的，在运行效率上远远低于C/C++这样经过编译后会生成machine native code的底层语言。而Mono则扩展了脚本语言的概念，所有Mono支持的编程语言都可以被AOT compiler便以为一种中间语言（IL: intermediate language)，最后在runtime通过JITcompiler编译为machine native code。这种语言的运行效率通常远胜于脚本语言，但同时又能享受到很多高级语言带给开发者的便利性。
Mono offers:
支持开发者使用包括C#在内的一系列语言来编写脚本
a native code generator to run your software as fast as possible
允许开发者在C#中调用底层代码
• InternalMethod provide access to raw C/C++ methods.
How does Mono support Game Developers?
游戏开发者是脚本技术的主要使用群体之一。以unity developer举例，性能攸关的代码大部分由unity引擎开发人员使用C/C++/Assembly编写并封装起来，而游戏开发人员负责的则是上层的游戏系统与玩法的实现。
而这种架构的问题在于，即使引擎的开发人员深入到最底层，充分利用好GPU,CPU的每一丝性能，游戏的最终性能仍然会被脚本语言的性能所束缚。
通过使用Mono所提供的脚本系统，比使用一般的解释型语言在性能的提升非常大。
The Mono Runtime
The mono runtime implements ECMA Common Language Infrastructure (CLI). It is a virtual machine that can be used to run ECMA and .NET applications.
The Mono runtime can be used as a stand-alone process, or it can be embedded into applications
ECMA CLI是一个规范，.NET Core CLR 和 mono runtime 都是这个规范的实现。

AOT Compilation
AOT的理念是允许开发者提前将代码编译为机器码以加快程序的启动速度，并且减少runtime时JIT compiler的工作量。
在 .NET 中负责这个功能的工具是 ngen.exe, it is a tool that improves the performance of managed applications. Ngen.exe creates native images, which are files containing compiled processor-specific machine code, and installs them into the native image cache on the local computer. The runtime can use native images from the cache instead of using the just-in-time (JIT) compiler to compile the original assembly.
Some features like Reflection.Emit and other forms of dynamic code generation are not support in this scenario.
