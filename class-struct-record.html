Reference Type and Value Type
A reference type is a type which has as its value a reference to the appropriate data rather than the data itself. Classes, interfaces, delegates, arrays are all reference types.
A value type directly contain the data, assignment of a value type involves actual data being copied.
Value types are immutable. 这意味着当我们为一个值类型变量赋值后，就再也无法在内存中对其进行修改，所有试图对值类型变量的修改都会在内存中生成一个新的实例。
值类型/引用类型变量的生命周期和存储方式
local variables are always stored on the stack
local variables means variables declared in a method
parameters count as local variables as well, unless they are passed with the ref keyword, that way they don’t get their own slot in the stack, but share a slot with the variable used in the calling code
local reference type variables are also stored on the stack。 但是注意，这个变量本身存储的只是一个引用，一串地址，而不是其所引用的对象，对象仍然存储在 managed heap 中。
Instance variables for a reference type are always on the heap， where the object itself "lives".
Instance variables for a value type与自己所属的值类型对象的存储方式一致。
如果我们在一个函数内实例化一个结构体，那么该结构体的成员变量就会被存储在栈中。
如果一个结构体是类的成员变量，那么这个结构体包括其所有的成员变量都会存储在堆中。
static variables are always stored on the heap, there is only one memory slot taken in total for each static variable of a type, no matter how many instances are created.
captured variables, 即闭包变量，在 C# 代码里以local variable的形式出现，但实际上会被编译为一个匿名类的 instance variable，因此与instance variable的生命周期相同。
Boxing and Unboxing
Boxing moves a value type variable to the heap, unboxing moves it back.
int a = 3; object boxing_a = a; //隐式boxing int b = 3; object boxing_b = (object)b; //显式boxing a = (int)boxing_a; //unboxing

Should we initialize a field at constructor or at declaration？
In C# assignment in declaration is performed before calling the superclass constructor, therefore they are almost the same. The most important rule is to be consistent in your practice.
What’s the difference between readonly, const, and static fields?
readonly field的值在对象的生命周期中无法被改变，只能在构造函数中被赋值
对于值类型字段来说，这意味着该字段的值无法改变，例如，如果一个int类型的只读字段被初始化为1，那么在对象的生命周期内我们无法将它的值修改为任何其他数字。
对于引用类型字段来说，由于字段的值是一个地址、一个指针，这意味着该字段所指向的内存地址不可变，但这个指针所指向的对象中的属性、字段仍然可以被修改。
编译器会在编译时直接将const所修饰的变量替换为变量的值，意味着const字段在运行时是不存在的，会被编译器优化掉。
static members are initialized with static constructor. .net runtime不保证static constructor在什么时间点会被执行，因此在其中不应该添加任何具有依赖性的代码

ref, in, out - 传参修饰符
ref
如果直接向函数中传入一个引用类型的参数，那么我们可以修改该引用类型变量内部的值，但无法为引用类型变量本身重新赋值，是该变量指向别的内存地址
如果我们使用ref关键字修饰该引用类型的参数，就可以将其重新赋值
in
in 关键字会导致按引用传递参数，但确保在函数中无法修改参数的内存地址。
对于值类型参数来说，无法为该参数重新赋值
对于引用类型变量来说，可以修改其内部字段的值，但同样也无法为其重新赋值
out
如果我们需要一个函数返回多个返回值，可以使用out修饰词，将返回值在参数列表中标明出来
使用ref关键词修饰的参数在传入函数之前必须被初始化，而使用out关键词修饰的参数则无需如此

Struct
class and record 如此方便，为什么我们仍然需要 struct ？
class/record对象被存放在堆内存中，对开发人员而言，这让我们可以更方便的管理它们的生命周期，缺点是有一定的性能开销。
游戏开发对性能的要求很高，能使用对象池缓存可复用对象就尽量使用对象池，能减少实例化和GC的次数就尽量减少。而对象的销毁一定会有gc的介入。
to reduce the work needed by the garbage collector, you can use the stack for smaller object, and structs are stored in stack
readonly struct
如果一个struct的内部状态在初始化后不需要被改变，可以将用readonly修饰符。compiler will make sure you don't add any members that change states(property with set operator; fields without readonly modifier)，并在编译时报错。

enum, flag 与位运算
在C#中，枚举类型背后所实现的值类型是整型的，因此我们可以为一个枚举类型中所有的枚举值用二进制赋值，从而实现位运算，这样我们就可以用一个枚举变量来保存一些状态。
我们可以把一般的枚举类型看成单选题，枚举变量在同一时间只能保存一个特定的枚举值，就像我们在做选择题是选择了A就不能选B。
用二进制赋值的枚举类型又被称为位枚举，可以将它看作多选题，我们可以同时选择A或B。我们只需要在枚举上添加 FlagAttribute，就可以使用很多拓展方法。
[flags] public enum Options { None = 0, // 0000_0000_0000_0000 A = 1 << 0, // 0000_0000_0000_0001 B = 1 << 1, // 0000_0000_0000_0010 C = 1 << 2, // 0000_0000_0000_0100 D = 1 << 3, // 0000_0000_0000_1000 All = -1 // 1111_1111_1111_1111 } var option = Options.A | Options.B; // 0000_0000_0000_0011 bool aSelected = option.HasFlag(Options.A); // true bool cSelected = option.HasFlag(Options.C); // false
