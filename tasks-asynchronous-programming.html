Why Async Programming is Important?
Many APIs with .NET offers both a synchronous and an asynchronous version.
而对于一个耗时较长的异步函数来说，开发者可以自行选择是否等待该异步函数，如果开发者选择不等待，那么caller function在异步函数的执行期间仍然可以继续向下执行。
Many APIs with .NET offers both a synchronous and an asynchronous version.
.NET 在不同的版本中提供了很多种异步编程模式。在C#5.0之前，这些异步模式都是基于函数回调的机制，在异步操作完成后执行回调函数。
C# 5.0 introduced the task-based async pattern, which is based on .NET 4 features, the task parallel library. With this pattern, an asynchronous method returns a Task, and you can use the await keyword to wait for the result.
A modern class for doing network requests with the task-based async pattern is HttpClient.GetAsync()
Task-Based Async Pattern
以上文提到过的实现了这个异步编程模式的HttpClient类为例，这个类中几乎所有的公共函数名都有Async的后缀，并且会返回一个Task。
By using the await keyword, the calling thread is not blocked, but the result variable response is only filled as soon as the Task returned from the async method is completed.
此时，Task.Status = RanToComplete; 表示任务已经被成功完成。
using System; using System.Net.Http; using System.Threading.Tasks; string uri = "https://www.google.ca/"; // by adding the 'using' keyword, HttpClient.Dispose() will be called // at the end of the variable scope using HttpClient client = new(); try { using HttpResponseMessage response = await client.GetAsync(uri); // HttpResponseMessage object has a IsSuccessStatusCode property to verify if the // response from the service was successful. if (response.IsSuccessStatusCode) { string html = await response.Content.ReadAsStringAsync(); Console.WriteLine(html[..200]); } else { Console.WriteLine($"status code: {response.StatusCode}"); } } catch (Exception e) { Console.WriteLine("error occurred"); }

The async and await keywords are compiler features. The compiler creates code by using functionality from the Task class.
.NET中的Task是基于多线程的，这就是为什么我们需要在unity中使用UniTask
为了方便理解，我们可以先实现一个辅助函数TraceThreadAndTask()用于打印一些与当前Task相关的信息。
Task.CurrentId returns the identifier of the task. Thread.CurrentThread.ManagedThreadId returns the identifier of the current thread
static void TraceThreadAndTask(string info) { string taskInfo = Task.CurrentId == null ? "no task" : "task " + Task.CurrentId; Console.WriteLine($"{info} in " + $"thread {Thread.CurrentThread.ManagedThreadId} " + $"and {taskInfo}" ); }

Calling an Asynchronous Method
We can call an asynchronous method by using the await keyword on the task that is returned.
async static void CallAsync() { TraceThreadAndTask($"Started {nameof(CallAsync)}"); string result = await GreetingAsync("Zack"); Console.WriteLine(result); TraceThreadAndTask($"ended {nameof(CallAsync)}"); } //output //Started CallAsync in thread 1 and no task //running GreetingAsync in thread 10 and task 8 //running Greeting in thread 10 and task 8 //Hello, Zack //ended CallAsync in thread 10 and no task
C#
当我们运行这个函数时，根据第一行输出可以看出：此时并没有任何正在执行的tasks。而异步函数GreetingAsync()则会使用一个与caller function不同的线程来执行一个task。非异步函数 Greeting() 则会在它的caller function，也就是GreetingAsync() 的线程上运行。
当异步函数的执行结束后，程序的控制权会重新回到 CallAsync()，但它会继续新的线程10上运行，而不会返回其一开始所在的线程1。
The await keyword make sure that the continuation happens after the task was completed, but it now uses a different thread.
Using the Awaiter
You can use the async keyword with any object that offers the GetAwaiter() method and returns an awaiter.
An awaiter implements the interface INotifyCompletion with the method OnCompleted() , it will be invoked when the task is completed.
public interface INotifyCompletion { void OnCompleted(Action continuation); }
