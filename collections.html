List
One feature of array is that its size is fixed. For resizable arrays, .NET offers the generic class List<T> . It implements IList, ICollection, IEnumerable, IList<T>, ICollection<T> and IEnumerable<T> interfaces.
List的非泛型版本是ArrayList，它内部所维护的数组是object类型的。
Capacity of List
using the default constructor of List creates an empty list of capacity 0. As soon as we add an element to the list, the capacity of the list will extend to 4. After that, when the size of the list exceeds its capacity, the capacity always doubles.
when the capacity of a list changes, the completed collection is reallocated to a new memory block. List会在内存中重新申请一块空间来存放新的内部数组，并将旧数组中的数据都通过Array.Copy()函数拷贝过来。
显然，频繁的reallocation会带来性能上的问题。因此，如果我们能够提前知道 List 中元素的个数，或者知道其上界，就可以通过构造函数声明你想要的List的初始容量，避免在List的使用过程中反复对其进行resize。
Get Rid of Unused Memory Space
当我们使用一个实例化的列表时，如果我们明确的知道我们不会继续往这个列表中田假日元素，就可以使用List.TrimExcess() 来清理其所占用的多余的内存空间。
Remove Elements
List.RemoveAt()可以通过传入待移除元素在 List 中的 index 来移除这个元素
List.Remove()则可以传入待移除元素的引用，让 List 通过循环的方式匹配列表中的元素并删除。
很明显，通过 index 的移除操作更加高效。通过待移除元素的引用的删除操作会遍历整个列表，找到相应元素的 index， 然后通过 index 移除该元素。

Linked List
在C#的实现中，LinkedList<T>是一个双向链表。与一般的链表相比，双向链表中的每个元素都会用一个额外的指针指向它之前的那个元素。
优点
当我们向链表中的任意一个位置添加一个新元素时，只需要改变其前后两个元素的指针所指向的位置O(1)，效率很高。而在基于数组实现的数据结构中，向数组中添加元素需要将所有 index 在其之后的元素向后挪动一个位置 O(n)。
缺点
LinkedList<T>的内部并不是使用数组实现的，因此，它不会在内存中开辟一大片连续的内存空间用于存放所有的元素，而是会在每个元素新加入时动态分配出一个新的内存空间来存放该元素。
因此，这不符合数据局部性(Data locality)的原则，相比数组和列表会触发更多的 cache miss。
同时，因为链表的实现机制，我们无法随机读取（random access）链表中的元素。假如我们想找访问链表中第n个元素，只能通过指针遍历的方式获取它。
另外，链表中的每个元素都需要两个额外的指针指向其在链表中的前后两个位置的元素，会产生额外的内存消耗。
LinkedList<int> linkedList = new(); //add head node LinkedListNode<int> head = linkedList.AddFirst(1); //add second node LinkedListNode<int> second = linkedList.AddAfter(head, 2); //add tail node LinkedListNode<int> tail = linkedList.AddLast(3); //traverse the list LinkedListNode<int>? node = head; while (node != null) { Console.WriteLine(node.Value); node = node.Next; }

Queue
FIFO: the item that is put first in the queue is read first.
Queue<T> implements IEnumerable<T>, ICollectionand IReadOnlyCollection . 没有实现 ICollection<T>的原因是这个接口声明了增删操作，但队列这种数据结构不应该支持这种操作。
对于队列而言，我们有进列(enqueue)和出列(dequeue)。
与List相似，Queue<T>内部维护了一个T类型的数组来储存队列中的元素。
它维护了队列头和对列尾在内部数组中的位置，因此，当tail到达数组的最后一个元素时，假如head并不是位于数组的第一个元素的位置，那么新加入的元素可以被加入到数组的开头，直到整个数组都被元素注满，这时我们必须进行扩容操作（grow）。
//队列的扩容操作 private void SetCapacity(int capacity) { T[] newarray = new T[capacity]; if (_size > 0) { if (_head < _tail) { Array.Copy(_array, _head, newarray, 0, _size); } else { Array.Copy(_array, _head, newarray, 0, _array.Length - _head); Array.Copy(_array, 0, newarray, _array.Length - _head, _tail); } } _array = newarray; _head = 0; _tail = (_size == capacity) ? 0 : _size; _version++; }

Set
Collections that contains only distinct items is known by the term set.
.NET core 基于两种不同的数据结构实现了两个set类，分别是HashSet<T>和SortedSet<T> 。它们都实现了ISet<T>接口。
HashSet是基于哈希表实现的，因此其中的元素是无序的。
SortedSet是基于红黑树设计的树结构，因此其中的元素是有序的。Set中仅包含对树的根节点root的引用。
ISet<T>
这个接口抽象出所有并查集的操作，例如：合并操作，判断当前集合是否为另一个集合的子集等。
public interface ISet<T> : ICollection<T>, IEnumerable<T>, IEnumerable { bool Add(T item); void UnionWith(IEnumerable<T> other); void IntersectWith(IEnumerable<T> other); void ExceptWith(IEnumerable<T> other); void SymmetricExceptWith(IEnumerable<T> other); bool IsSubsetOf(IEnumerable<T> other); bool IsSupersetOf(IEnumerable<T> other); bool IsProperSupersetOf(IEnumerable<T> other); bool IsProperSubsetOf(IEnumerable<T> other); bool Overlaps(IEnumerable<T> other); bool SetEquals(IEnumerable<T> other); }
