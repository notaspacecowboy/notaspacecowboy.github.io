Shallow Copy vs Deep Copy
由于Array是引用类型，因此企图通过直接赋值的方式将一个数组变量中的内容拷贝到另一个数组变量是不可行的，我们实现的仅仅是指针赋值，赋值完成后，会新增一条指向内从中数组地址的指针。
如果我们想要真的在内存中开辟一片新的空间存放一个一摸一样的数组，则需要使用Array.Clone()
当我们调用该函数时，如果数组中存放的是值类型数据，则会将所有值都拷贝到新的数组中；但如果数组中存放的是引用类型，则只会拷贝引用。这种拷贝被称作浅拷贝（shallow copy）。
与之相对的深拷贝（deep copy），则会完全拷贝数组中的所有元素而非引用，不管是引用类型数据还是值类型数据。
C#中没有内置的函数帮助我们实现数组的深拷贝，我们需要自己遍历整个数组来实例并初始化每个元素。

yield: C#中的迭代器
当使用C#中提供的 yield return / yield break 语法时，编译器会生成一个状态机来对一个集合进行迭代。
yield return 返回集合中的一个元素，并将游标推进到下个元素的位置。
yield break 结束遍历。
当我们在函数中使用yield语句时，C#编译器会自动生成一个实现了IEnumerable和IEnumerator接口的类，并为我们返回它的实例
这个自动生成的类对 IEnumerator.MoveNext() 的实现中通过 switch 语句实现了一个简单的状态机。每次调用IEnumerator.MoveNext()都会将state + 1，假设源代码中使用了3次yield return，那么编译器会自动生成4个state，对应被3个yield return语句分成4部分的函数语句。
Span
System.Span 是C#中提供的一个内置结构体，使用一种类型安全且内存安全的方式来封装一片连续的内存。通常用于数组和字符串。
它可以指向任何内存，不管是堆内存还是栈内存。当然，因为System.Span是一个结构体，因此它的实例本身会存储在栈中。
以substring()这个函数为例，由于在C#中，string类型是immutable的，因此substring()函数的调用会导致在内存中重新开辟一片新的内存空间，用于存放该函数所返回的子串。而Span最强大也是最实用的特性就是可以通过它提供的Span.Slice()函数截取数组或字符串中的一部分，而不用像Array.Copy()或string.Substring()一样拷贝数据，并在内存中开辟新的空间。（由于Span是值类型的结构体，因此它的实例化会在栈内存中进行）

Array Pools
在某些应用场景中，我们可能需要频繁的创建并销毁数组，这会给 GC 带来很大的压力。这时我们可以使用 C# 内置的 ArrayPool<T> ，它是一个abstract class，意味着我们无法通过构造函数生成它的实例，它帮助我们静态的管理了一个Array类型的对象池。我们可以通过这个类所提供的 API 向其请求 / 返回数组。
//对象池初始化 //定义了一个数组的极限长度，以及该对象池的每个bucket中最多有几个数组 //The array pool uses multiple buckets for faster access to arrays //when many arrays are used. Arrays of similar sizes are kept in the same bucket // as long as possible, and the maximum number of arrays is not reached. ArrayPool<int> customPool = ArrayPool<int>.Create(maxArrayLength: 40000, maxArraysPerBucket: 10); //You can also use a predefined shared pool by accessing the S //hared property of the ArrayPool<T> class ArrayPool<int> sharedPool = ArrayPool<int>.Shared; //用rent函数从对象池中“借”一个数组。 //minLength只规定了这个被借出的数组长度的最小值，具体借出的数组长度由 ArrayPool 根据自身情况分配。 //The shared pool keeps arrays with at least 16 elements. //The element count of the managed arrays always doubles //for example, 16, 32, 64, 128, 256, 512, 1024, 2048, 4096, 8192 elements, and so on. int[] intArray = sharedPool.Rent(50);

BitArray
BitArray is reference type that indeed contains an array of int, where for every 32 bits a new integer is used. BitArray defines Count and Length properties, an indexer, a SetAll method to set all the bits according to the parameters passed, a Not method to inverse the bits, as well as And, Or, Xor methods for binary AND, binary OR, and exclusive OR.
