Implicit and Explicit Conventions
当C# code 经过编译器后，会被编译为 IL 代码，IL代码中使用的是强类型系统，这是 .NET 很多服务和功能模块的基础。因此，C# 也是一个强类型语言。对于强类型语言来说，类型转换相对复杂。
Implicit Conversions
编译器只有在确保没有数据会被丢失的情况下才会允许隐式转换而不报错。
For example, by conventing from an int to byte, you were potentially losing 3 bytes of data. Thus the compiler does not let you do that unless you explicitly specify that's what you want to do.
nullable value types do not implicitly convert to non-nullable value types
