Handling Errors
C#'s mechanism for handling error conditions enables you to provide custom handling for each type of error condition and to separate the code that identifies errors from the code that handles them.
In C#, an exception is an object created (or thrown) when a particular exceptional error condition occurs. This object contains information that can help identify the problem.

Exceptions Filter
C# 6 was enhanced to support exception filters. A catch block runs only if the filter returns true. You can have different catch blocks that act differently when catching different exception types.

What Happens If an Exception Isnâ€™t Handled?
Sometimes an exception might be thrown, but there is no catch block in your code that is able to handle that kind of exception.
In this situation, .NET runtime would catch it. The .NET runtime has effectively placed the entire program inside another huge try blockâ€”it does this for every .NET program.
This try block has a catch handler that can catch any type of exception. If this catch handler catch an exception, the application will be terminated promptly and the user will see a dialog that complains that your code has not handled the exception and provide some details of it.
try { if (2 > 1) { throw new FormatException("456"); } } //our code failed to catch the exception, so it will be handled by .NET runtime catch (IndexOutOfRangeException e) { Console.WriteLine(123); }
