Handling Errors
C#'s mechanism for handling error conditions enables you to provide custom handling for each type of error condition and to separate the code that identifies errors from the code that handles them.
In C#, an exception is an object created (or thrown) when a particular exceptional error condition occurs. This object contains information that can help identify the problem.
异常处理会带来性能上的影响。最好不要用异常处理的方式来解决一些可能在程序中频繁发生的错误。
.NET中比较常见的异常类型
System.Exception是 .NET 中所有异常的基类，在 .NET 的设计理念中，针对每种异常都应该生成一个继承自 System.Exception 的子类实例。
ArgumentException
当函数中某个参数的值不符合函数对其的预期时被抛出
NotSupportedException
it is thrown when a method is not supported
for example, from classes that implement interfaces but do not implement all the members of the interface
You should not invoke methods that throw this exception, so you should not handle this exception. This exception gives good information during development time to change your code.
StackOverflowException
it is thrown by runtime when the area of memory allocated for the stack is full
通常会在某个函数陷入死循环，或者递归层数过多时被抛出。
这通常是非常致命的异常，因为大部分情况下这会导致程序直接闪退。因此指望程序自己通过trycatch语句来处理这种异常是不现实的。
NullReferenceException
当我们试图访问一个赋值为空的类对象的成员时被抛出。
IndexOutOfRangeException
当我们试图访问一个非法的数组/集合坐标时被抛出。
Catching Exceptions
异常的捕捉和处理可以分成三个步骤。
try blocks encapsulate the code that forms part of the normal operation of your program and that might encounter some serious error conditions.
catch blocks encapsulate the code dealing with the various error conditions that your code might have encountered by working through any of the code in the accompanying try block. This block could also be used for logging errors.
finally blocks encapsulate the code that cleans up any resources or takes any other action that you normally want handled at the end of a try or catch block. finally中的代码在trycatch执行完毕后一定会被触发: It's executed on a successful return of the try block and also executed if an exception is thrown.
try { //code for normal execution } catch { //error handling } finally { //clean up, optional }
C#
As soon as the application encounters a throw statement inside a try block, it immediately looks for the catch block associated with that try block.
you can have throwstatements that are nested in several method calls inside the try block, but the same try block continues to apply even as execution flow enters these other methods.
当程序抛出异常时，它会沿着函数运行对栈不断向上回溯，直到找到可以处理这个被抛出的异常的catch block。
During this process, all the local variables in the intermediate method calls will correctly go out of scope.
Exceptions Filter
C# 6 was enhanced to support exception filters. A catch block runs only if the filter returns true. You can have different catch blocks that act differently when catching different exception types.
public static void ThrowWithErrorCode(int code) { throw new MyCustomException("Error in Foo") { ErrorCode = code }; } try { ThrowWithErrorCode(405); } //只有在异常类型相符且异常实例的ErrorCode property的值等于405时执行该catch语句 //这种写法很类似泛型类对于泛型的限制条件 catch (MyCustomException ex) when (ex.ErrorCode == 405) { Console.WriteLine($"Exception caught with filter {ex.Message} " + $"and {ex.ErrorCode}"); } catch (MyCustomException ex) { Console.WriteLine($"Exception caught {ex.Message} and {ex.ErrorCode}"); }

What Happens If an Exception Isn’t Handled?
Sometimes an exception might be thrown, but there is no catch block in your code that is able to handle that kind of exception.
In this situation, .NET runtime would catch it. The .NET runtime has effectively placed the entire program inside another huge try block—it does this for every .NET program.
This try block has a catch handler that can catch any type of exception. If this catch handler catch an exception, the application will be terminated promptly and the user will see a dialog that complains that your code has not handled the exception and provide some details of it.
try { if (2 > 1) { throw new FormatException("456"); } } //our code failed to catch the exception, so it will be handled by .NET runtime catch (IndexOutOfRangeException e) { Console.WriteLine(123); }

Caller Information
在做异常处理时，如果能够知道错误发生的具体位置（文件名，类名，行数）显然会对Debug有很大帮助。我们可以通过在函数的参数列表内假如一些被特定Attribute所修饰的可选参数来获取这些有用的信息。
public void Log([CallerLineNumber] int line = -1, [CallerFilePath] string path = default, [CallerMemberName] string name = default) { Console.WriteLine($"Line {line}"); Console.WriteLine(path); Console.WriteLine(name); Console.WriteLine(); }
