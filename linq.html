Deferred Query Execution: LINQ 的延迟执行特性
Overview
LINQ的全称是Language Integrated Query，顾名思义，指的是一套集成在语言之中的数据查询模块。
C#定义了一些SQL中常用的关键字，如 from, where, select, orderby, 这使得我们可以像写 SQL query 一样写 LINQ query。
from r in NBA.GetChampions() where r.Country == "Brazil" orderby r.Wins descending select r;

C#
LINQ是基于 IEnumerator + Extension method 实现的，因此上面code中的写法只是C#中的语法糖，最终会被编译器转换为 extension method 的格式。
LINQ 在 System.Linq 这个命名空间下的 Enumerable 类中为 IEnumerable<T> 接口实现了多个拓展方法，因此我们可以用LINQ语句查询任何实现了该接口的集合变量 - List，Stack，Queue，Dictionary …
//这是 Enumerable 中的其中一个拓展方法。 public static IEnumerable<TSource> Where<TSource>( this IEnumerable<TSource> source, Func<TSource, bool> predicate) { if (source == null) throw Error.ArgumentNull(nameof (source)); if (predicate == null) throw Error.ArgumentNull(nameof (predicate)); switch (source) { case Enumerable.Iterator<TSource> _: return ((Enumerable.Iterator<TSource>) source).Where(predicate); case TSource[] _: return (IEnumerable<TSource>) new Enumerable.WhereArrayIterator<TSource>((TSource[]) source, predicate); case List<TSource> _: return (IEnumerable<TSource>) new Enumerable.WhereListIterator<TSource>((List<TSource>) source, predicate); default: return (IEnumerable<TSource>) new Enumerable.WhereEnumerableIterator<TSource>(source, predicate); } }

Deferred Query Execution: LINQ 的延迟执行特性
由于 所有LINQ 函数的返回值都是一个实现了IEnumerable<T>接口的对象，因此LINQ的数据查询斌不是在查询创建时执行的，而是在遍历时执行，,也就是在enumerator的MoveNext()方法被调用的时候执行。
