Delegates
Delegates are the .NET variant of addresses to method, it is an object-oriented type-safe pointer to one or multiple methods.

C#
注意，当我们向一个委托中传入函数时，只要传入该函数名，而不能加函数后的括号。否则C#编译器会认为我们希望在这里调用该函数，因此，编译器会自动将函数的返回值传入给委托。
Delegate 特性：Type Safe
通过委托来封装函数的好处之一在于：函数的签名必须严格遵守委托类型的限制。委托变量不关心这个被传入的函数的访问权限（private, public, protected），也不关心这个函数属于哪个类或哪个对象，更不关心它是 instance method 还是 static method。他只管维护函数指针，并在合适的时候通过指针调用其内部保存的所有函数。
指向多个函数的委托 - Multicast delegates
Multicast delegates实际上是通过在该委托变量的内部维护一个含有多个委托的集合来实现的。
为什么我们不应该向委托中传入相互依赖的函数？
委托变量中维护着多个函数指针时，函数的调用顺序严格意义上来说是不确定的。
委托变量中维护者多个函数指针时，函数是一个接着一个被链式调用的，如果其中一个函数的调用抛出了异常，整个调用链就会停止，后续应该被调用的函数不再会被调用。
在不确定委托函数是否会抛出异常的情况下，可以通过Delegate.GetInvocationList()获取 Multicast delegate中维护的委托集合，然后通过遍历的方式手动调用其中的每个函数，根据需要使用try catch做异常处理。这同样也是 Power Chord 中实现异步委托的方式。
Anonymous Method
使用匿名函数的优点在于: 我们不必为一个只会被使用一次的函数费心的设计它的函数名，有些函数仅仅被用于为委托变量赋值，使用匿名函数可以简化代码的复杂度。
使用匿名函数并不会让我们的代码跑得更快，编译器仍然会为我们定义一个随机命名的函数。
在匿名函数的基础上更进一步，就是我们常用的 Lambda Expression。
string mid = ", middle part, "; Func<string, string> anonDel = delegate(string param) { param += mid; param += " and this was added to the string."; return param; } ampmDel("Start of string");

Lambda Expressions
When we use it?
One way lambda expressions are used is to assign code—using a lambda expression—to a parameter.
You can use lambda expressions whenever you have a delegate parameter type.
注意点
Lambda Expression 通常都是用于对一个 Delegate 变量进行赋值，又因为委托是类型安全的，其严格限定了函数的签名，因此在Lambda expression中我们不需要显式的写出参数和返回值的类型。
Closure - 闭包
What is closure?
With lambda expressions, you can access variables outside the block of the lambda expression. This is known as closure.
内层的函数可以引用包含在它外层的函数的变量，即使外层函数的执行已经终止。
What issues can be caused by closures?
//假设在这里我们定义了一个委托变量f，并用Lambda表达式对其初始化 int someVal = 5; Func<int, int> f = x => x + someVal; //很显然，此时调用f(x)时，会返回x+5 //但假如我们在调用f(x)之前先修改一下someVal这个变量的值，此时再调用f(x), 返回值也会发生变化 f(0); //return 0+5=5 someVal = 10; f(0); //return 0+10=10 //更容易令人迷惑的地方在于，如果我们在Lambda表达式内部修改了someVal这个闭包变量的值， //外部代码一样也会访问到被修改后的最新值。 Func<int, int> f = x => { x += someVal; someVal = 0; return x; } f(0); //return 0 + 5 = 5; log(someVal); //print 0

闭包问题在C#中是如何产生的？
在C#中,闭包只是编译器玩的花招而已,它仍然没有脱离.NET对象生命周期的规则，闭包变量会被直接封装到编译器自动生成的匿名类中，变成类的一个公共字段，从而保证了闭包变量的生命周期不会随外部函数的调用结束而结束。
对于闭包变量的修改会在闭包函数内外都会生效，因为函数内部和外部所使用的闭包变量都会被替换成这个匿名类中的公共字段。
//上面代码片段所产生的匿名类 public class AnonymousClass { public int _someVal; public AnonymousClass(int someVal) => _someVal = someVal; public int AnonymousMethod(int x) => x + _someVal; } //Using the lambda expression and invoking the method creates an instance of the //anonymous class and passes the value of the variable from the time when the //call is made. var instance = new AnonymousClass(someVal); instance.AnonymousMethod(x);

Events
What is Events in C#?
Events are based on delegates, and it offers a publish / subscribe mechanism to delegates.
事件的本质是委托字段的一个包装器，它对委托字段的访问起到限制作用；事件对外界隐藏了委托实例的大部分功能，仅暴露添加/移除事件处理器的功能。
Event 的整个工作流程至少需要牵扯到两个概念，即事件的发起方和事件的监听方：事件的发起方在事件发生时invoke事件，事件的监听方注册事件的监听事件，在事件发生时被动的执行callback。
注意事项
在使用 event 关键字时，官方推荐最好不要使用 custom delegate types, 而是使用C#提供的 EventHandler<TEventArgs>类型。
EventHandler是预定义的委托类型，其一共接收两个参数，第一个参数是object类型，表示信息的发送方，通常用this指针为其赋值即可。第二个参数是TEventArgs类型，通常被当做数据类型使用，用于传入事件的具体信息。
TEventArgs没有任何类型上的限制，但最好继承自 EventArgs
public class CarInfoEventArgs : EventArgs { public string Car { get; } public CarInfoEventArgs(string car) => Car = car; } public class CarDealer { public event EventHandler<CarInfoEventArgs>? NewCarCreated; public void CreateANewCar(string car) { Console.WriteLine($"Car Dealear, new car {car}"); } private void RaiseNewCarCreated(string car) { NewCarCreated?.Invoke(this, new CarInfoEventArgs(car)); } }
